# 女娲 · Skill 造人术

> 「写不进去的那部分，才是你真正的护城河。但写得进去的部分，已经足够强大。」

捕捉的是 **HOW they think**，不是 WHAT they said。
蒸馏的不是人，是**认知操作系统**。

---

## Phase 0 — 入口分流

收到用户输入后，先判断路径：

| 用户输入 | 路径 |
|---------|------|
| 明确的人名/主题 | **直接路径** — 确认人名 + 聚焦方向，直接进入 Phase 1 |
| 模糊的需求/困惑 | **诊断路径** — 1-2 轮追问 → 推荐 2-3 个候选 → 用户选择 → Phase 1 |

**诊断路径追问原则**：
- 最多问 2 轮，不要变成问卷调查
- 需求场景够清晰时直接推荐，不追问
- 推荐候选时说清楚「这个人的哪个框架匹配你的需求」以及「他的局限在哪」

**检查是否已有 Skill**：先用 `recall_skill("[人名]")` 搜索。已有则直接激活，不重新蒸馏。

用户确认人选后，立即告知启动并进入 Phase 1。

## Phase 1 — 多维信息采集

**顺序执行 6 个调研方向**，每个方向用 `fetch_url` 搜索相关页面并记录关键发现。
每完成一个方向，在工作记录中简记核心发现，供后续 Phase 2 使用。

| # | 方向 | 搜索目标 | 关键提取点 |
|---|------|---------|-----------|
| 1 | **著作** | 书籍 / 长文 / newsletter | 反复出现 ≥3 次的核心论点、自创术语、推荐书单 |
| 2 | **对话** | 播客 / 采访 / AMA / 长视频 | 即兴类比、被追问时的回答方式、改变立场的瞬间、拒绝回答的问题 |
| 3 | **表达** | Twitter/X / 微博 / 短文 | 高频词句式、争议立场、幽默方式、公开辩论记录 |
| 4 | **他者视角** | 传记 / 批评文章 / 同行评价 | 外部观察到的行为模式、主要批评点、与同行的差异 |
| 5 | **决策记录** | 重大决策 / 转折点 / 争议行为 | 决策背景与逻辑、事后反思、言行一致/不一致案例 |
| 6 | **时间线** | 从出道到现在的完整轨迹 | 关键里程碑、思想转折点、**最近 12 个月动态**（防过时）|

**信息源黑名单**（永远排除）：知乎、微信公众号、百度百科、百度知道。

**中文人物优先来源**：B 站原始视频（非搬运号）、小宇宙/喜马拉雅播客音频、
36 氪、晚点 LatePost、财新、极客公园、少数派、本人著作/微博。

**信息可信度分级**（记录时标注）：
- 一手：本人写的 / 本人说的（最高）
- 二手：他人记录本人言论（中等）
- 推断：基于行为模式的推测（低，需标注）

**调研完成后，展示摘要给用户确认再继续：**
```
┌─────────────────────────────────────────────┐
│ 调研摘要 · [人名]                            │
├──────────┬──────────┬────────────────────────┤
│ 方向     │ 来源数量  │ 核心发现               │
├──────────┼──────────┼────────────────────────┤
│ 著作     │ N 篇     │ ...                    │
│ 对话     │ N 段     │ ...                    │
│ 表达     │ N 条     │ ...                    │
│ 他者视角  │ N 篇     │ ...                    │
│ 决策记录  │ N 个     │ ...                    │
│ 时间线   │ 完整/不完整│ 最新动态: ...         │
├──────────┼──────────┼────────────────────────┤
│ 矛盾点   │ N 处     │ ...                    │
│ 信息不足  │ 无/有    │ ...                    │
└──────────┴──────────┴────────────────────────┘
```

用户确认 OK → Phase 2。
用户觉得某维度不够 → 补充调研后再继续。

## Phase 2 — 框架提炼

### 2.1 心智模型（3-7 个）

从 6 个方向的素材中扫描所有候选论点，对每个候选执行**三重验证**：

**验证 1 · 跨域复现**
同一个思维框架出现在此人讨论的 ≥2 个不同领域？
（例：纳瓦尔的「杠杆」概念在财富/个人成长/职业选择中均出现 → 真信念）

**验证 2 · 有生成力**
用这个模型能推断此人对**新问题**的可能立场？
（例：芒格的「逆向思维」→ 面对「如何成功」会先想「如何确保失败」→ 可生成推断）

**验证 3 · 有排他性**
不是所有聪明人都这样想，体现此人的独特视角？
（「反脆弱」是塔勒布的，不是通用智慧 → 有区分度）

> 三重通过 → **心智模型**
> 通过 1-2 重 → **决策启发式**（见 2.2）
> 通过 0 重 → 丢弃

**每个心智模型记录格式**：
```
### [模型名称]
[一句话描述这个镜片如何看世界]

来源证据：[场景 A] / [场景 B]（至少 2 个不同领域）
应用：面对 [X 类问题] 时，用这个模型…
失效条件：当 [Y 情况] 时，这个模型不适用
```

取 top 3-7 个（按排他性强度排序）。宁少勿多——3 个深刻远好于 10 个浅薄。

### 2.2 决策启发式（5-10 条）

此人做判断时的快速规则，可表述为「如果 X，则 Y」，有具体案例支撑。
比心智模型更具体，适用于特定场景的行动规则。

### 2.3 表达 DNA

| 维度 | 提取内容 |
|------|---------|
| 句式偏好 | 长句/短句、结论先行/先铺垫、疑问密度 |
| 词汇特征 | 高频词、自创术语、禁忌词（从不使用的词）|
| 确定性表达 | 「我不确定」型 vs「很明显/一定」型 |
| 幽默方式 | 讽刺/自嘲/荒诞对比/冷幽默/不幽默 |
| 引用习惯 | 爱引谁、偏好什么类型的证据（数据/故事/类比）|
| 节奏感 | 先结论还是先铺垫、转折密度 |

### 2.4 价值观、反模式与内在张力

- **价值观**：3-5 条核心排序（用「A > B」格式）
- **反模式**：此人明确反对的行为/思维方式
- **内在张力**：价值观之间的冲突——这是深度的来源，**必须保留矛盾，不要和稀泥**

矛盾类型：
- 时间性（早期说 A，后来说 B）→ 记录演化轨迹，以近期为主
- 领域性（工作中主张 X，生活中主张 Y）→ 分领域记录
- 本质性张力（价值观内在冲突）→ 标注为「核心张力」

### 2.5 诚实边界（必写）

- 不能预测面对全新问题的反应
- 不能替代此人的创造力和直觉
- 公开表达 ≠ 真实想法
- 信息截止至本次调研日期
- [其他信息不足的维度，如有]

**提炼完成后，展示摘要给用户确认：**
```
提炼摘要：
- 心智模型: N 个 — [列出名称]
- 决策启发式: N 条
- 表达 DNA 关键特征: [3 个核心特征]
- 核心张力: [N 对，各一句话描述]
- 诚实边界: N 条
```

用户确认 OK → Phase 3。用户觉得某个模型不对 → 回到 Phase 2 调整。

```markdown

author: "nuwa-distilled"
version: "1.0.0"
---

# [人名] · 认知操作系统

> [此人最有代表性的一句话，用此人语气]

**核心定位**：[此人是谁，一句话，用第三人称]

---

## 身份

- **句式**：[特征]
- **高频词**：[词1] / [词2] / [词3]
- **禁忌词**：[从不使用的表达]

```

### 保存步骤

1. **保存为 Skill**（持久化且立即可用）：
   `remember_skill("[person-slug]-perspective", <完整skill内容>)`
   保存后 `use_skill("[person-slug]-perspective")` 可直接加载

2. **不要手动写 SKILL.md**：
   不要用 `write_file(.../SKILL.md)` 创建 Skill。Skill 属于运行时能力资产，不是 Files 面板里的生成文件；`remember_skill` 会写入正确目录并自动刷新 registry。

**保存完成后告知用户：**
```
✅ 蒸馏完成：[人名]

调用方式：
  use_skill("[person-slug]-perspective")  — 直接加载完整指令

示例：
  > use_skill("munger-perspective")
  > 用芒格的视角分析这笔投资
```

## Phase 4 — 质量验证

构建完成后执行 3 项检查，完成后展示验证结果：

**4.1 已知测试**
选 2 个此人公开表态过的问题，用 Skill 回答，核对方向一致性。
- 一致 → 模型有效
- 偏离 → 回到 Phase 2 调整心智模型权重

**4.2 边缘测试**
选 1 个此人没有公开讨论过但相关的问题，用 Skill 推断。
- 期望：「基于[模型X]的推断，可能…但不确定」（适度不确定性）
- 不应该：斩钉截铁给出此人不会说的结论

**4.3 风格测试**
用 Skill 写一段 100 字分析，检查：
- 有此人的表达特征？
- 不像通用 AI 鸡汤？
- 不是原话拼凑，而是框架在运转？

**通过标准**：

| 检查项 | 通过 | 不通过信号 |
|--------|------|-----------|
| 心智模型数量 | 3-7 个，每个有来源证据 | <3 或 >10 |
| 每个模型有失效条件 | 明确写出 | 只写优点，无局限 |
| 表达 DNA 辨识度 | 读 100 字能认出是谁 | 像通用 ChatGPT |
| 诚实边界 | ≥3 条具体局限 | 只有「不能替代本人」 |
| 内在张力 | ≥2 对矛盾 | 观点高度一致（太假）|
| 一手来源占比 | >50% | 主要靠二手转述 |

验证通过 → 交付完成。
不通过 → 标注薄弱环节，回 Phase 2 迭代（最多 2 轮，超过则标注局限后交付当前最优版本）。

## 特殊场景处理

### 信息不足时

| 情况 | 处理 |
|------|------|
| 单维度来源不足 | 标注「此维度信息有限，以下为推测」|
| 全面信息匮乏（<10 条来源）| 提前告知用户，心智模型减至 2-3 个，扩大诚实边界篇幅 |
| 信息相互矛盾无法判断 | 并列呈现，让用户自行判断 |
| 此人刻意不公开某方面 | 注明「此人对这个话题保持沉默」|

### 用户蒸馏自己
当用户说「蒸馏我自己」「帮我做一个我的 Skill」时：
1. 无法从公开渠道搜索，需要用户提供素材
2. 引导用户提供：个人文章/博客、录制过的视频、写过的决策备忘录、自我描述
3. Phase 1 的 6 个调研方向改为分析用户提供的素材
4. 特别注意「自我认知偏差」——可以追问身边人的评价

### 更新已有 Skill
用户说「更新 XX 的 Skill」时：
1. 用 `recall_skill("[人名]")` 读取现有 Skill，找到「信息截止」日期
2. 只重跑方向 2（最新对话）+ 方向 5（最新决策）+ 方向 6（时间线更新）
3. 对比：强化现有模型 → 补充案例；与现有矛盾 → 标注变化更新；新模式出现 → 考虑增加
4. 增量更新，不重写整个 Skill

### 主题 Skill（非人物）
当输入是主题而非人名时（如「价值投资」「产品克制」「反脆弱决策」）：
- Phase 1：先搜索该主题的 3-5 个核心人物/流派，再分人物方向调研
- Phase 2.1：提炼「领域共识框架」+ 「各家分歧」
- Phase 2.3：不模拟特定人物语气，使用中性专业表达
- Phase 3 格式：去掉身份卡，改为「框架概览」+「流派对比」

## 品味守则

遇到判断困难时回看：

| 原则 | 说明 |
|------|------|
| 长文 > 金句 | 3000 字 essay 比 50 条推文更揭示思维结构 |
| 争议 > 共识 | 最被争议的观点最能揭示独特性 |
| 变化 > 固定 | 改变立场的地方比一直坚持的更有信息量 |
| 行为 > 宣言 | 实际决策 > 访谈中说的话 |

**绝不做**：
- 编造此人没说过的话
- 把通用道理包装成此人的「独特见解」
- 忽略负面评价和争议
- 在信息不足时强行生成高置信度结论
